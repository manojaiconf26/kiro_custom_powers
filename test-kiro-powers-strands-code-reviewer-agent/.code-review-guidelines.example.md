# Team Code Review Guidelines

> Copy this file to `.code-review-guidelines.md` in your project root and
> customize it for your team. The code reviewer agent will automatically
> pick it up.

## Language & Framework Standards

- **Python**: Follow PEP 8. Use type hints on all public functions. Prefer
  `pathlib.Path` over `os.path`. Use `dataclasses` or `pydantic` for data containers.
- **TypeScript**: Strict mode enabled. No `any` types except in test fixtures.
  Prefer `interface` over `type` for object shapes.
- **API Design**: All REST endpoints must return consistent error envelopes:
  `{ "error": { "code": "...", "message": "..." } }`.

## Architecture Rules

- Services must not import from other services directly. Use the event bus or
  shared contracts in `libs/contracts/`.
- Database access is only allowed through repository classes. No raw SQL in
  handlers or controllers.
- All new endpoints require an OpenAPI schema update in `docs/openapi.yaml`.

## Security Policies

- Never log PII fields (email, phone, SSN). Use the `redact()` helper from
  `libs/security/`.
- All user input must be validated with schema validation before processing.
- Secrets must come from environment variables or AWS Secrets Manager — never
  hardcoded, never in config files.
- Authentication tokens must have a maximum TTL of 1 hour.

## Testing Requirements

- All new features require unit tests with ≥80% branch coverage.
- Integration tests are required for any new API endpoint.
- Test names must follow the pattern: `test_<unit>_<scenario>_<expected>`.

## Error Handling

- Use custom exception classes from `libs/exceptions/`. Do not raise generic
  `Exception` or `RuntimeError`.
- All external API calls must have retry logic with exponential backoff.
- Failed operations must log structured error context (request ID, user ID,
  operation name) — but never PII.

## Git & PR Conventions

- Commit messages follow Conventional Commits: `feat:`, `fix:`, `chore:`, etc.
- PRs must not exceed 400 lines of diff (excluding generated files).
- Breaking changes require a migration guide in the PR description.
